XML - язык разметки для хранения и передачи данных

Ответы на вопросы по языку XML на основе материалов Лекции 10

Основы XML

1. Что такое XML? Лекция 10

XML (eXtensible Markup Language) — это язык разметки, созданный для хранения, транспортировки и обмена данными между различными системами. XML является подмножеством SGML (Standard Generalized Markup Language).

Из Лекции 10: "Открытый язык разметки (Extensible Markup Language, XML) представляет собой подмножество SGML и полностью описывается в спецификации от 10 февраля 1998 (XML 1.0)."

Ключевые характеристики XML:

  • Расширяемый — можно создавать собственные теги
  • Структурированный — данные организованы в древовидную структуру
  • Самодокументируемый — имена тегов описывают содержание
  • Независимый от платформы — работает на любых системах
  • Поддерживается всеми современными языками программирования

Пример XML-документа:

<?xml version="1.0" encoding="UTF-8"?>
<!-- Это комментарий в XML -->
<книга>
  <название язык="ru">Война и мир</название>
  <автор>Лев Толстой</автор>
  <год издания="1869" тип="первое"/>
</книга>

2. Чем является первая строка в коде XML-документа? Лекция 10

Первая строка в XML-документе называется XML-декларацией (XML declaration) или прологом документа. Она сообщает парсеру XML, что это XML-документ и какой версии.

Из Лекции 10: "Первые строки до корневого элемента называются прологом документа и сообщает о том, что документ XML и какой версии."

Синтаксис XML-декларации:

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>

Атрибуты XML-декларации:

  • version — версия XML (обязательный атрибут)
  • encoding — кодировка символов (например, UTF-8, Windows-1251)
  • standalone — указывает, зависит ли документ от внешних источников

Примеры XML-деклараций:

<!-- Минимальная декларация -->
<?xml version="1.0"?>

<!-- С указанием кодировки -->
<?xml version="1.0" encoding="UTF-8"?>

<!-- Полная декларация -->
<?xml version="1.0" encoding="Windows-1251" standalone="yes"?>

3. Назовите правила синтаксиса XML-документа. Лекция 10

XML-документы должны следовать строгим правилам синтаксиса:

Из Лекции 10: В лекции представлен раздел "Правила синтаксиса XML" с перечислением 6 основных правил.

Основные правила синтаксиса XML:

1. Закрывающие теги

Все элементы должны иметь закрывающий тег. Пустые элементы могут использовать сокращенную форму: <empty/>

2. Регистрозависимость

Имена элементов регистрозависимы: <Book> и <book> — разные элементы

3. Корректная вложенность

Элементы должны быть правильно вложены: <a><b></b></a>

4. Корневой элемент

Документ должен содержать ровно один корневой элемент

5. Кавычки для атрибутов

Значения атрибутов должны заключаться в кавычки (одинарные или двойные)

6. Комментарии

Комментарии помещаются между <!-- и -->, но не перед XML-декларацией

Пример правильного и неправильного синтаксиса:

Правильно:

<?xml version="1.0"?>
<book>
  <title lang="en">XML Guide</title>
  <author>John Doe</author>
  <year>2023</year>
</book>

Неправильно:

<?xml version="1.0"?>
<book>
  <title lang=en>XML Guide  ← Нет кавычек
  ← Нет закрывающего тега
  <author>John Doe
  <year>2023</YEAR> ← Разный регистр
</book>

4. Какие должны быть имена элементов? Лекция 10

Имена элементов XML должны следовать определенным правилам:

Из Лекции 10: "XML элементы должны следовать следующим правилам написания имен..."

Правила именования элементов XML:

Допустимые имена:

  • <book>
  • <book_title>
  • <Book1>
  • <книга> (кириллица)
  • <_book>
  • <book-title>

Недопустимые имена:

  • <1book> (начинается с цифры)
  • <xml> (начинается с "xml")
  • <book title> (пробел внутри)
  • <book> (спецсимвол)
  • <book> (спецсимвол)

Полные правила из лекции:

  1. Имена могут содержать буквы, цифры и другие символы
  2. Имена не могут начинаться с цифры или символа пунктуации
  3. Имена не могут начинаться с сочетания "xml" (или XML, или Xml и т.п.)
  4. Имена не могут содержать пробельные символы

5. Как вы понимаете правило соблюдения корректной вложенности? Лекция 10

Правило корректной вложенности означает, что элементы XML должны быть правильно вложены друг в друга — внутренние элементы должны полностью находиться внутри внешних элементов, без пересечений.

Из Лекции 10: "XML элементы должны соблюдать корректную вложенность" (правило синтаксиса №3).

Примеры вложенности:

Корректная вложенность:

<library>
  <book>
    <title>XML Guide</title>
    <author>
      <name>John</name>
      <surname>Doe</surname>
    </author>
  </book>
</library>

Каждый элемент полностью находится внутри родительского элемента.

Некорректная вложенность:

<library>
  <book>
    <title>XML Guide
      <author> ← Начинается внутри title
    </title> ← Закрывается до author
      <name>John</name>
    </author> ← Закрывается после title
  </book>
</library>

Элементы пересекаются — <author> начинается внутри <title>, но заканчивается после него.

Аналогия: Представьте матрешки — меньшая кукла должна быть полностью внутри большей. Нельзя, чтобы одна матрешка начинала открываться внутри другой, но заканчивалась вне её.

6. Какие элементы являются корневыми в XML-документе? Лекция 10

Корневой элемент — это элемент самого верхнего уровня в XML-документе, который содержит все остальные элементы. В XML-документе может быть только один корневой элемент.

Из Лекции 10: "XML документ должен содержать корневой документ" (правило синтаксиса №4).

Примеры корневых элементов:

<?xml version="1.0"?>
<!-- <catalog> - корневой элемент -->
<catalog>
  <book id="1">
    <title>XML Basics</title>
    <author>John Smith</author>
  </book>
  <book id="2">
    <title>Advanced XML</title>
    <author>Jane Doe</author>
  </book>
</catalog>

Ошибка: несколько корневых элементов

<?xml version="1.0"?>
<book>Книга 1</book>  ← Первый корневой элемент
<book>Книга 2</book>  ← Второй корневой элемент - ОШИБКА!

Такой документ не является well-formed (правильно сформированным).

7. Для чего необходима валидация XML-документа? Лекция 10

Валидация XML-документа — это процесс проверки документа на соответствие определенной структуре и правилам, описанным в DTD (Document Type Definition) или XML Schema.

Из Лекции 10: "DTD описывает: Какие элементы могут присутствовать в документе; Вхождение элементов; Возможные атрибуты элементов; и т.д."

Цели валидации XML:

1. Проверка структуры

Убедиться, что документ содержит все необходимые элементы в правильном порядке

2. Обеспечение совместимости

Гарантировать, что разные системы будут одинаково интерпретировать документ

3. Обнаружение ошибок

Найти ошибки в данных до их обработки приложением

4. Контроль качества данных

Обеспечить целостность и корректность передаваемых данных

Типы валидных XML-документов:

Из Лекции 10: "Первое правило для валидного XML документа то, что он должен быть синтаксически верным. Второе правило — валидный XML документ должен соответствовать определенному типу документов."

  1. Well-formed (правильно сформированный) — следует базовым правилам синтаксиса XML
  2. Valid (валидный) — следует правилам, определенным в DTD или XML Schema

XML Schema и DTD

8. В чем заключается разница между простыми и комплексными типами элементов в XML Schema? Лекция 10

В XML Schema элементы разделяются на простые и комплексные типы в зависимости от их содержимого.

Сравнение простых и комплексных типов:

Простой тип (simpleType)

  • Содержит только текстовые данные
  • Не может иметь дочерних элементов
  • Может иметь атрибуты (только если определен как complexType с simpleContent)
  • Примеры: строки, числа, даты
<xs:element name="title" type="xs:string"/>
<xs:element name="price" type="xs:decimal"/>
<xs:element name="year" type="xs:integer"/>

Комплексный тип (complexType)

  • Может содержать дочерние элементы
  • Может содержать атрибуты
  • Может содержать смешанный контент (текст + элементы)
  • Определяет структуру элемента
<xs:element name="book">
  <xs:complexType>
    <xs:sequence>
      <xs:element name="title" type="xs:string"/>
      <xs:element name="author" type="xs:string"/>
    </xs:sequence>
    <xs:attribute name="id" type="xs:integer"/>
  </xs:complexType>
</xs:element>

9. Перечислите преимущества XML Schema перед DTD. Лекция 10

XML Schema имеет несколько важных преимуществ перед DTD:

Из Лекции 10: В разделе "Преимущества XML Schema" перечислены ключевые преимущества.

Преимущества XML Schema:

1. Написана на XML

XML Schema сама является XML-документом, что позволяет обрабатывать её теми же инструментами

2. Поддержка типов данных

Имеет богатую систему типов данных (строки, числа, даты, пользовательские типы)

3. Пространства имен

Полная поддержка XML Namespaces для избежания конфликтов имен

4. Расширяемость

Легко расширяется и наследуется

5. Богатые возможности

Поддержка ограничений, шаблонов, уникальности, ключей и ссылок

6. Современный стандарт

Рекомендован W3C как современная замена DTD

Сравнение DTD и XML Schema:

DTD

<!ELEMENT book (title, author, year)>
<!ELEMENT title (#PCDATA)>
<!ELEMENT author (#PCDATA)>
<!ELEMENT year (#PCDATA)>
<!ATTLIST book
  id CDATA #REQUIRED
  category CDATA #IMPLIED>

• Ограниченные типы данных

• Нет поддержки пространств имен

XML Schema

<xs:element name="book">
  <xs:complexType>

</xs:element>

10. Какие типы элементов XML Schema вы знаете? Лекция 10

XML Schema поддерживает различные типы элементов для определения структуры документа:

Основные типы элементов в XML Schema:

1. Простые элементы

Содержат только текстовые данные определенного типа

<xs:element name="name" type="xs:string"/>

2. Комплексные элементы

Содержат другие элементы и/или атрибуты

<xs:element name="person">
  <xs:complexType>...</xs:complexType>
</xs:element>

3. Глобальные элементы

Определены на корневом уровне схемы, могут использоваться многократно

<xs:element name="address" type="addressType"/>

4. Локальные элементы

Определены внутри complexType, видны только в его контексте

<xs:complexType name="bookType">
  <xs:sequence>
    <xs:element name="title" type="xs:string"/>
  </xs:sequence>
</xs:complexType>

11. Что означает xs:element? Лекция 10

xs:element — это элемент XML Schema, который используется для объявления элемента в XML-документе. Префикс xs указывает на пространство имен XML Schema.

Использование xs:element:

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">

  <!-- Простое определение элемента -->
  <xs:element name="title" type="xs:string"/>

  <!-- Комплексное определение элемента -->
  <xs:element name="book">
    <xs:complexType>
      <xs:sequence>
        <xs:element ref="title"/> <!-- Ссылка на элемент выше -->
        <xs:element name="author" type="xs:string"/>
      </xs:sequence>
    </xs:complexType>
  </xs:element>

</xs:schema>

Атрибуты xs:element:

  • name — имя элемента в XML-документе
  • type — тип данных элемента
  • ref — ссылка на глобальное определение элемента
  • minOccurs — минимальное количество вхождений (по умолчанию 1)
  • maxOccurs — максимальное количество вхождений (по умолчанию 1, unbounded — неограниченно)

12. Поясните, что означает xs:complexType. Лекция 10

xs:complexType — элемент XML Schema, который определяет комплексный тип данных. Комплексные типы могут содержать дочерние элементы, атрибуты и смешанный контент.

Примеры xs:complexType:

complexType с дочерними элементами:

<xs:complexType name="bookType">
  <xs:sequence>
    <xs:element name="title" type="xs:string"/>
    <xs:element name="author" type="xs:string"/>
    <xs:element name="year" type="xs:integer"/>
  </xs:sequence>
</xs:complexType>

complexType с атрибутами:

<xs:complexType name="personType">
  <xs:simpleContent>
    <xs:extension base="xs:string">
      <xs:attribute name="age" type="xs:integer"/>
      <xs:attribute name="gender" type="xs:string"/>
    </xs:extension>
  </xs:simpleContent>
</xs:complexType>

Типы содержимого complexType:

  • xs:sequence — элементы в строгом порядке
  • xs:choice — выбор одного из элементов
  • xs:all — элементы в любом порядке
  • xs:simpleContent — только текст + атрибуты
  • xs:complexContent — наследование от другого complexType

13. В чем заключается предназначение xs:sequence? Лекция 10

xs:sequence — элемент XML Schema, который указывает, что дочерние элементы должны появляться в строго определенной последовательности.

Пример использования xs:sequence:

<xs:complexType name="bookType">
  <xs:sequence>
    <xs:element name="title" type="xs:string"/>
    <xs:element name="author" type="xs:string"/>
    <xs:element name="year" type="xs:integer"/>
    <xs:element name="publisher" type="xs:string"
               minOccurs="0"/> <!-- Необязательный -->
  </xs:sequence>
</xs:complexType>

Правила xs:sequence:

  • Элементы должны следовать в том же порядке, как указано в sequence
  • Каждый элемент может иметь атрибуты minOccurs и maxOccurs
  • По умолчанию каждый элемент должен появляться ровно один раз
  • sequence может содержать другие sequence, choice или all

Пример валидного и невалидного XML для этой схемы:

Валидный XML:

<book>
  <title>XML Guide</title>
  <author>John Doe</author>
  <year>2023</year>
  <!-- publisher может отсутствовать -->
</book>

Невалидный XML:

<book>
  <author>John Doe</author> ← Должен быть после title!
  <title>XML Guide</title>
  <year>2023</year>
</book>

14. Для чего используется xs:attribute? Лекция 10

xs:attribute — элемент XML Schema, который определяет атрибут для элемента. Атрибуты предоставляют дополнительную информацию об элементах.

Примеры использования xs:attribute:

<xs:complexType name="bookType">
  <xs:sequence>
    <xs:element name="title" type="xs:string"/>
    <xs:element name="author" type="xs:string"/>
  </xs:sequence>
  <!-- Определение атрибутов -->
  <xs:attribute name="id" type="xs:ID" use="required"/>
  <xs:attribute name="language" type="xs:string"
                 default="en"/>
  <xs:attribute name="category" type="xs:string"
                 use="optional"/>
</xs:complexType>

Атрибуты xs:attribute:

  • name — имя атрибута
  • type — тип данных атрибута
  • use — обязательность (required, optional, prohibited)
  • default — значение по умолчанию
  • fixed — фиксированное значение

Соответствующий XML-документ:

<book id="b123" language="en" category="technical">
  <title>XML Schema Guide</title>
  <author>Jane Smith</author>
</book>

DTD и сущности

15. Каково предназначение ENTITY? Лекция 10

ENTITY в DTD — это сущность, которая представляет собой именованное значение или фрагмент контента, который можно повторно использовать в XML-документе.

Из Лекции 10: "Инструкция ENTITY используется для определения сущностей в DTD с целью их использования как в связанном с DTD XML-документе, так и собственно в DTD."

Типы сущностей ENTITY:

Внутренние сущности:

<!ENTITY company "Microsoft Corporation">
<!ENTITY copyright "© 2023 Все права защищены">
<!ENTITY email "info@example.com">

Определяются прямо в DTD, содержат текстовые значения.

Внешние сущности:

<!ENTITY chapter1 SYSTEM "chapter1.xml">
<!ENTITY logo SYSTEM "logo.png" NDATA png>
<!ENTITY disclaimer PUBLIC 
  "-//Example//TEXT Disclaimer//EN"
  "disclaimer.txt">

Ссылаются на внешние файлы или ресурсы.

Использование сущностей в XML:

<?xml version="1.0"?>
<!DOCTYPE document [
  <!ENTITY author "Иван Иванов">
  <!ENTITY company "ООО 'Технологии'">
  <!ENTITY year "2023">
]>
<document>
  <header>
    <title>Документация</title>
    <author>&author;</author>
    <company>&company;</company>
  </header>
  <footer>
    &company; &copy; &year;
  </footer>
</document>

16. Что такое сущности? Каким образом вы использовали сущности? Лекция 10

Сущности (entities) в XML — это именованные фрагменты данных, которые можно повторно использовать в документе. Они действуют как макросы или переменные.

Из Лекции 10: "Сущность (entity) является заместителем содержания, которую можно однажды объявить и многократно использовать почти в любом месте документа."

Способы использования сущностей:

1. Для повторяющегося текста

<!ENTITY legal "Все права защищены">
...
<footer>&legal;</footer>

2. Для специальных символов

<price>100&euro;</price>
<text>5&gt;3</text>

3. Для внешних файлов

<!ENTITY chapter1 SYSTEM "ch1.xml">
...
<chapter>&chapter1;</chapter>

4. Для параметрических сущностей

<!ENTITY % blocks "(p|div|section)">
<!ELEMENT content %blocks;>

Практический пример использования сущностей:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE report [
  <!-- Определение сущностей -->
  <!ENTITY company "TechSolutions Inc.">
  <!ENTITY address "123 Main St, New York">
  <!ENTITY phone "+1 (555) 123-4567">
  <!ENTITY email "info@techsolutions.com">
  <!ENTITY copyright "© 2023 TechSolutions Inc.">
]>
<report>
  <header>
    <company>&company;</company>
    <contact>
      <address>&address;</address>
      <phone>&phone;</phone>
      <email>&email;</email>
    </contact>
  </header>
  <content>
    <!-- Содержимое отчета -->
  </content>
  <footer>
    &copyright;
  </footer>
</report>

Преимущества использования сущностей:

  • Централизованное управление повторяющимися данными
  • Упрощение обновления информации (изменение в одном месте)
  • Сокращение объема документа
  • Улучшение читаемости и поддерживаемости

17. Какие встроенные сущности вы знаете? Лекция 10

Встроенные (предопределенные) сущности XML — это специальные символы, которые имеют заранее определенные имена в XML. Они используются для представления символов, которые имеют специальное значение в XML.

Из Лекции 10: В разделе "Встроенные (предопределенные) сущности XML" перечислены основные сущности.

Основные встроенные сущности XML:

Сущность Символ Описание Пример
&lt; < Знак "меньше" 5 &lt; 10 → 5 < 10
&gt; > Знак "больше" 10 &gt; 5 → 10 > 5
&amp; & Амперсанд Smith &amp; Sons → Smith & Sons
&apos; ' Апостроф Don't → Don't
&quot; " Кавычка &quot;Hello&quot; → "Hello"

Пример использования встроенных сущностей:

<math>
  <expression>
    x &lt; y &amp; y &gt; z
  </expression>
  <description>
    Формула: a&sup2; + b&sup2; = c&sup2;
  </description>
  <quote>
    Он сказал: &quot;Hello, World!&quot;
  </quote>
</math>

Важно: Символы <, > и & всегда должны быть экранированы в XML, так как они имеют специальное значение (теги и начало сущностей).

18. Для чего используется ATTLIST? Лекция 10

ATTLIST (Attribute List) — это инструкция DTD, которая используется для объявления атрибутов элементов и определения их свойств.

Из Лекции 10: "Инструкция ATTLIST используется для перечисления и объявления всех атрибутов, которые могут принадлежать элементу."

Синтаксис ATTLIST:

<!ATTLIST элемент
  атрибут1 тип значение_по_умолчанию
  атрибут2 тип значение_по_умолчанию
  ...>

Пример использования ATTLIST:

<!ELEMENT book (title, author, year)>
<!ELEMENT title (#PCDATA)>
<!ELEMENT author (#PCDATA)>
<!ELEMENT year (#PCDATA)>

<!-- Объявление атрибутов для элемента book -->
<!ATTLIST book
  id ID #REQUIRED
  category CDATA #IMPLIED
  language CDATA "en"
  available (yes|no) "yes"
>

Типы значений по умолчанию:

  • #REQUIRED — атрибут обязателен
  • #IMPLIED — атрибут не обязателен
  • #FIXED "значение" — атрибут имеет фиксированное значение
  • "значение" — значение по умолчанию

19. Раскройте суть валидного XML-документа. Какие типы вы знаете? Лекция 10

Валидный XML-документ — это документ, который не только является well-formed (правильно сформированным), но и соответствует правилам, определенным в DTD или XML Schema.

Из Лекции 10: "Первое правило для валидного XML документа то, что он должен быть синтаксически верным. Второе правило — валидный XML документ должен соответствовать определенному типу документов."

Типы XML-документов:

1. Well-formed (правильно сформированный)

  • Следует базовым правилам синтаксиса XML
  • Имеет один корневой элемент
  • Все теги правильно закрыты
  • Элементы правильно вложены
  • Атрибуты в кавычках

2. Valid (валидный)

  • Является well-formed
  • Соответствует DTD или XML Schema
  • Содержит все обязательные элементы
  • Элементы в правильном порядке
  • Атрибуты имеют допустимые значения

Пример валидного XML-документа с DTD:

<?xml version="1.0"?>
<!DOCTYPE catalog [
  <!ELEMENT catalog (book+)>
  <!ELEMENT book (title, author, year)>
  <!ATTLIST book id ID #REQUIRED>
  <!ELEMENT title (#PCDATA)>
  <!ELEMENT author (#PCDATA)>
  <!ELEMENT year (#PCDATA)>
]>
<catalog>
  <book id="b1">
    <title>XML Basics</title>
    <author>John Doe</author>
    <year>2023</year>
  </book>
</catalog>

Этот документ является валидным, потому что:

  1. Он well-formed (правильный синтаксис XML)
  2. Соответствует правилам в DTD (есть все обязательные элементы и атрибуты)

20. Что означает следующая запись <!ENTITY name "Hello, world!">? Лекция 10

Эта запись определяет внутреннюю сущность в DTD с именем name и значением "Hello, world!".

Разбор записи:

<!ENTITY name "Hello, world!">
  • <!ENTITY — начало объявления сущности
  • name — имя сущности (будет использоваться как &name;)
  • "Hello, world!" — значение сущности
  • > — конец объявления

Пример использования:

<?xml version="1.0"?>
<!DOCTYPE greeting [
  <!ENTITY hello "Hello, world!">
]>
<greeting>
  <message>&hello;</message>
  <repeat>
    Снова: &hello;
  </repeat>
</greeting>

После обработки парсером XML будет выглядеть так:

<greeting>
  <message>Hello, world!</message>
  <repeat>
    Снова: Hello, world!
  </repeat>
</greeting>

Парсер заменяет &hello; на значение сущности "Hello, world!".

21. Поясните следующую запись <!ELEMENT to (#PCDATA)>. Лекция 10

Эта запись определяет элемент в DTD с именем to, который может содержать только текстовые данные (Parsed Character DATA).

Из Лекции 10: В примерах представлена именно эта запись: <!ELEMENT to (#PCDATA)>

Разбор записи:

<!ELEMENT to (#PCDATA)>
  • <!ELEMENT — начало объявления элемента
  • to — имя элемента
  • (#PCDATA) — тип содержимого: Parsed Character Data (текст, который будет обработан парсером)
  • > — конец объявления

Пример использования:

<?xml version="1.0"?>
<!DOCTYPE note [
  <!ELEMENT note (to, from, message)>
  <!ELEMENT to (#PCDATA)>
  <!ELEMENT from (#PCDATA)>
  <!ELEMENT message (#PCDATA)>
]>
<note>
  <to>Иван Иванов</to>
  <from>Петр Петров</from>
  <message>Привет! Как дела?</message>
</note>

Другие типы содержимого в DTD:

  • EMPTY — элемент не может содержать ничего (например: <br/>)
  • ANY — элемент может содержать любой контент
  • (child1, child2, ...) — последовательность дочерних элементов
  • (child1|child2|...) — выбор одного из дочерних элементов

22. Какие параметры и значения имеет инструкция ATTLIST? Лекция 10

Инструкция ATTLIST имеет следующую структуру параметров:

Из Лекции 10: "Сначала указывается имя элемента (или элементов), к которому относится список атрибутов. Затем для всех атрибутов по очереди указывается имя, обязательность и символьные данные, допустимые в качестве значения."

Полный синтаксис ATTLIST:

<!ATTLIST имя-элемента
  имя-атрибута1 тип-атрибута значение-по-умолчанию
  имя-атрибута2 тип-атрибута значение-по-умолчанию
  ...>

Типы атрибутов в DTD:

Тип Описание Пример
CDATA Текстовые данные name CDATA #IMPLIED
ID Уникальный идентификатор id ID #REQUIRED
IDREF Ссылка на ID ref IDREF #IMPLIED
ENTITY Имя сущности logo ENTITY #IMPLIED
NMTOKEN Имя (без пробелов) code NMTOKEN #REQUIRED
(val1|val2|...) Перечисление значений status (active|inactive) "active"

Примеры значений по умолчанию:

<!ATTLIST book
  <!-- Обязательный атрибут -->
  id ID #REQUIRED

  <!-- Необязательный атрибут -->
  category CDATA #IMPLIED

  <!-- Атрибут с значением по умолчанию -->
  language CDATA "en"

  <!-- Атрибут с фиксированным значением -->
  version CDATA #FIXED "1.0"

  <!-- Атрибут с перечислением значений -->
  status (new|used|damaged) "new"
>

23. Назовите параметры, которые имеет инструкция ELEMENT. Лекция 10

Инструкция ELEMENT в DTD имеет следующие параметры:

Из Лекции 10: "Сначала инструкция объявляет имя элемента, а затем определяет, какое содержимое допустимо в элементе."

Синтаксис инструкции ELEMENT:

<!ELEMENT имя-элемента содержимое>

Типы содержимого элемента:

1. EMPTY

Элемент не содержит содержимого

<!ELEMENT br EMPTY>

В XML: <br/>

2. ANY

Элемент может содержать любой контент

<!ELEMENT document ANY>

3. (#PCDATA)

Только текстовое содержимое

<!ELEMENT title (#PCDATA)>

4. (дочерние-элементы)

Определяет структуру дочерних элементов

<!ELEMENT book (title, author, year)>

Модификаторы в содержимом элементов:

<!-- Обязательный элемент (по умолчанию) -->
<!ELEMENT book (title, author, year)>

<!-- Необязательный элемент (?) -->
<!ELEMENT book (title, author?, year)>

<!-- Ноль или более раз (*) -->
<!ELEMENT book (title, author*, year)>

<!-- Один или более раз (+) -->
<!ELEMENT catalog (book+)>

<!-- Выбор одного из элементов (|) -->
<!ELEMENT contact (phone|email|address)>

<!-- Группировка элементов -->
<!ELEMENT book (title, (author+|editor+), year)>

<!-- Смешанное содержимое -->
<!ELEMENT paragraph (#PCDATA|emphasis|link)*>

24. Что находится на http://www.w3.org/2001/XMLSchema? Лекция 10

На http://www.w3.org/2001/XMLSchema находится пространство имен XML Schema, которое содержит определения всех элементов и типов данных для создания XML-схем.

Значение этого URL:

Это не веб-страница для просмотра в браузере, а URI (Uniform Resource Identifier), который идентифицирует пространство имен XML Schema, определенное консорциумом W3C.

Использование в XML Schema:

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"
               targetNamespace="http://www.example.com/myschema"
               elementFormDefault="qualified">

  <!-- Здесь xs: относится к пространству имен XML Schema -->
  <xs:element name="book" type="bookType"/>

  <xs:complexType name="bookType">
    <xs:sequence>
      <xs:element name="title" type="xs:string"/>
      <xs:element name="author" type="xs:string"/>
    </xs:sequence>
  </xs:complexType>

</xs:schema>

Что содержит пространство имен XML Schema:

  • Все элементы схемы: xs:element, xs:complexType, xs:sequence и т.д.
  • Простые типы данных: xs:string, xs:integer, xs:date
  • Ограничения и фацеты: xs:minLength, xs:maxInclusive
  • Аннотации: xs:annotation, xs:documentation

25. Каким образом можно объявить DTD? Как расшифровывается аббревиатура?

DTD (Document Type Definition) — определение типа документа. Это набор правил, который определяет структуру и содержание XML-документа.

Из Лекции 10: "Document Type Definition (DTD) — это язык описания структуры XML-документа, который используется для проверки грамматики XML-документа и его соответствия определенному DTD."

Способы объявления DTD:

1. Внутреннее DTD (встроенное в XML)

<?xml version="1.0"?>
<!DOCTYPE note [
  <!ELEMENT note (to,from,heading,body)>
  <!ELEMENT to (#PCDATA)>
  <!ELEMENT from (#PCDATA)>
  <!ELEMENT heading (#PCDATA)>
  <!ELEMENT body (#PCDATA)>
]>
<note>
  <to>Иван</to>
  <from>Петр</from>
  <heading>Напоминание</heading>
  <body>Не забудь встречу!</body>
</note>

2. Внешнее DTD (ссылка на файл)

<?xml version="1.0"?>
<!DOCTYPE note SYSTEM "note.dtd">
<note>
  <to>Иван</to>
  <from>Петр</from>
  <heading>Напоминание</heading>
  <body>Не забудь встречу!</body>
</note>

Файл note.dtd:

<!ELEMENT note (to,from,heading,body)>
<!ELEMENT to (#PCDATA)>
<!ELEMENT from (#PCDATA)>
<!ELEMENT heading (#PCDATA)>
<!ELEMENT body (#PCDATA)>

3. Публичное DTD (стандартное)

<?xml version="1.0"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" 
               "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
  <head>
    <title>Пример XHTML</title>
  </head>
  <body>
    <p>Привет, мир!</p>
  </body>
</html>

Расшифровка аббревиатуры DTD:

  • Document — документ
  • Type — тип
  • Definition — определение

DTD = Document Type Definition = Определение Типа Документа
