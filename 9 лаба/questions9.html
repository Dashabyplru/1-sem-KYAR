Модульные сетки и Grid-верстка

Ответы на вопросы по системам модульной верстки и CSS Grid

Модульные сетки

1. Что представляет из себя модульная сетка?

Модульная сетка — это система организации контента на веб-странице, основанная на использовании колонок и рядов. Она представляет собой невидимую структуру, которая помогает дизайнерам и разработчикам:

  • Распределять контент на странице
  • Создавать последовательный и гармоничный дизайн
  • Обеспечивать отзывчивость (адаптивность) сайта
  • Упрощать процесс верстки
  • Соблюдать визуальную иерархию

Ключевые характеристики модульных сеток:

  • Деление на колонки (обычно 12, 16 или 24)
  • Межколоночные промежутки (гаттеры)
  • Горизонтальные ряды
  • Возможность объединения колонок

2. Поясните суть системы модульной верстки Skeleton.

Skeleton — это легковесный CSS-фреймворк для создания адаптивных сеток. Его суть заключается в предоставлении простой и понятной системы из 12 колонок для построения макетов веб-страниц.

Основные особенности Skeleton:

  • Использует 12-колоночную сетку
  • Имеет фиксированную максимальную ширину 960px
  • Автоматически адаптируется под мобильные устройства
  • Минимальный размер (всего около 400 строк CSS)
  • Не включает встроенные компоненты UI (только сетка)

Демонстрация 12-колоночной сетки Skeleton:

12 колонок (100%)
6 колонок (50%)
6 колонок (50%)
4 колонки (33%)
4 колонки (33%)
4 колонки (33%)
3 колонки (25%)
6 колонок (50%)
3 колонки (25%)

3. Каким образом можно использовать систему Skeleton?

Систему Skeleton можно использовать несколькими способами:

  1. Скачать и подключить файлы Skeleton: 
    <link rel="stylesheet" href="skeleton.css">
  2. Использовать через CDN: 
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/skeleton/2.0.4/skeleton.min.css">
  3. Встроить в проект и настроить под свои нужды

Базовый шаблон использования Skeleton:

<div class="container">
  <div class="row">
    <div class="one column">Одна колонка</div>
    <div class="eleven columns">Одиннадцать колонок</div>
  </div>
  <div class="row">
    <div class="two columns">Две колонки</div>
    <div class="ten columns">Десять колонок</div>
  </div>
</div>

4. Что означают стили в файле skeleton.css?

Файл skeleton.css содержит несколько ключевых групп стилей:

  1. Сброс и базовая типографика - нормализация стилей, базовые стили для текста, кнопок, форм
  2. Сеточная система - стили для контейнера, строк и колонок 
    .container {
  width: 100%;
  max-width: 960px;
  margin: 0 auto;
}

.row {
  display: flex;
  flex-wrap: wrap;
}

.column, .columns {
  width: 100%;
  padding: 0 15px;
}

/* Классы для разных размеров колонок */
.one.column { width: 8.33%; }
.two.columns { width: 16.66%; }
/* и так далее до twelve.columns */
  3. Медиа-запросы - адаптивные стили для мобильных устройств
  4. Утилитарные классы - вспомогательные стили для выравнивания, отображения и т.д.

5. Назовите этапы реализации системы модульной верстки Skeleton.

Реализация системы Skeleton включает следующие этапы:

  1. Подключение Skeleton - добавление CSS-файла в проект
  2. Создание контейнера - обертка с классом .container
  3. Создание строк - добавление элементов с классом .row
  4. Определение колонок - добавление колонок с классами .one, .two, ..., .twelve и .column или .columns
  5. Распределение контента - размещение контента внутри колонок
  6. Адаптация - проверка и корректировка отображения на разных устройствах

Пример реализации:

<!-- 1. Подключение Skeleton -->
<link rel="stylesheet" href="skeleton.css">

<!-- 2. Создание контейнера -->
<div class="container">
  
  <!-- 3. Создание строки -->
  <div class="row">
    
    <!-- 4. Определение колонок -->
    <div class="four columns">
      <!-- 5. Распределение контента -->
      <h3>Заголовок</h3>
      <p>Текст в колонке</p>
    </div>
    
    <div class="eight columns">
      <h3>Другой заголовок</h3>
      <p>Больше текста в широкой колонке</p>
    </div>
    
  </div>
</div>

CSS Grid верстка

6. Что такое grid-верстка?

CSS Grid — это мощная система двумерной верстки, которая позволяет создавать сложные макеты с помощью строк и колонок. В отличие от Flexbox (одномерная система), Grid дает полный контроль над обоими измерениями одновременно.

Ключевые особенности CSS Grid:

  • Двумерное позиционирование (строки и колонки одновременно)
  • Точное размещение элементов в любом месте сетки
  • Простое создание сложных, асимметричных макетов
  • Контроль над перекрытием элементов
  • Встроенные возможности для адаптивного дизайна

Базовый пример Grid-верстки:

Элемент 1
Элемент 2
Элемент 3
Элемент 4
Элемент 5
Элемент 6
.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 80px 120px 80px;
  gap: 10px;
}

7. Для чего необходимо свойство grid-template-areas?

Свойство grid-template-areas позволяет задавать имена для областей сетки и определять их расположение с помощью визуального шаблона. Это делает код более читаемым и упрощает позиционирование элементов.

Пример использования grid-template-areas:

Header
Sidebar
Main Content
Footer
.grid-container {
  display: grid;
  grid-template-columns: 1fr 2fr;
  grid-template-rows: 100px 300px 100px;
  grid-template-areas: 
    "header header"
    "sidebar content"
    "footer footer";
}

.header {
  grid-area: header;
}

.sidebar {
  grid-area: sidebar;
}

.content {
  grid-area: content;
}

.footer {
  grid-area: footer;
}

8. С какой целью используется свойство grid-template-columns?

Свойство grid-template-columns определяет количество, размеры и структуру колонок в Grid-контейнере. Оно позволяет задавать:

  • Количество колонок
  • Ширину каждой колонки
  • Гибкие единицы измерения (fr, auto, minmax())
  • Повторяющиеся паттерны

Примеры использования:

/* 3 колонки равной ширины */
grid-template-columns: 1fr 1fr 1fr;

/* 4 колонки с разной шириной */
grid-template-columns: 100px 1fr 2fr auto;

/* Повторяющийся паттерн */
grid-template-columns: repeat(3, 1fr);

/* Смешанные значения */
grid-template-columns: 200px repeat(2, 1fr) 100px;

9. Для чего применяется свойство grid-area?

Свойство grid-area выполняет две основные функции:

  1. Присвоение имени области - для использования с grid-template-areas 
    .element {
  grid-area: header;
}
  2. Сокращенное задание позиции - объединяет grid-row-start, grid-column-start, grid-row-end, grid-column-end 
    /* grid-area: row-start / column-start / row-end / column-end */
.element {
  grid-area: 1 / 1 / 3 / 4; /* Занимает строки 1-2 и колонки 1-3 */
}

10. С какой целью используется свойство grid-row?

Свойство grid-row определяет положение элемента по вертикали (в каких строках сетки он расположен). Это сокращенное свойство, которое объединяет grid-row-start и grid-row-end.

Примеры использования grid-row:

/* Элемент занимает строки с 1 по 3 */
.element {
  grid-row: 1 / 3;
}

/* Элемент начинается со строки 2 и заканчивается автоматически */
.element {
  grid-row: 2 / auto;
}

/* Элемент занимает 2 строки, начиная с текущей позиции */
.element {
  grid-row: span 2;
}

11. Что означает grid-template-columns: 20px 20px 20px;?

Запись grid-template-columns: 20px 20px 20px; означает создание Grid-контейнера с тремя колонками, каждая шириной 20 пикселей.

Визуализация:

1
2
3
4
5
6
.grid-container {
  display: grid;
  grid-template-columns: 20px 20px 20px;
  grid-template-rows: 40px 40px;
  gap: 10px;
}

Результат: создается сетка из 3 колонок по 20px каждая и 2 строк по 40px каждая, всего 6 ячеек.

12. Поясните запись grid-row-end: 4;

Запись grid-row-end: 4; означает, что элемент заканчивается на линии сетки номер 4. В CSS Grid линии нумеруются начиная с 1 (или с -1, если считать с конца).

Важно понимать: В Grid линии — это границы между ячейками, а не сами ячейки. Линия 1 находится перед первой строкой/колонкой, линия 2 — между первой и второй и т.д.

/* Элемент занимает пространство от линии 2 до линии 4 */
.element {
  grid-row-start: 2;
  grid-row-end: 4;
  /* Или в сокращенной форме: grid-row: 2 / 4; */
}

Это означает, что элемент будет занимать строки 2 и 3 (потому что он начинается на линии 2 и заканчивается на линии 4).

13. Для чего используется свойство grid-gap?

Свойство grid-gap (или его современные аналоги gap, row-gap, column-gap) задает расстояние между строками и колонками в Grid-контейнере.

Варианты использования:

/* Одинаковый отступ между строками и колонками */
grid-gap: 20px;

/* Разные отступы для строк и колонок */
grid-gap: 10px 20px; /* row-gap column-gap */

/* Современный синтаксис */
gap: 20px;
row-gap: 10px;
column-gap: 20px;

Примечание: grid-gap — устаревшее свойство, рекомендуется использовать gap, которое также работает с Flexbox.

15. С какой целью применяется свойства align-self и justify-self?

Свойства align-self и justify-self используются для выравнивания отдельного элемента внутри своей ячейки Grid:

  • justify-self - выравнивание по горизонтали (вдоль строки)
    • start - к началу ячейки
    • end - к концу ячейки
    • center - по центру
    • stretch - растянуть (по умолчанию)
  • align-self - выравнивание по вертикали (вдоль колонки)
    • Принимает те же значения, что и justify-self

Демонстрация align-self и justify-self:

align-self: start
align-self: center
align-self: end
justify-self: start
justify-self: center
justify-self: end
.item1 {
  align-self: start;    /* Выравнивание по верху ячейки */
}

.item2 {
  align-self: center;   /* Выравнивание по центру по вертикали */
}

.item3 {
  align-self: end;      /* Выравнивание по низу ячейки */
}

.item4 {
  justify-self: start;  /* Выравнивание по левому краю */
}

.item5 {
  justify-self: center; /* Выравнивание по центру по горизонтали */
}

.item6 {
  justify-self: end;    /* Выравнивание по правому краю */
}
